SELF JOIN

The Self join is mainly preferred to join a table with itself. Thus means that each row of the table is combined with itself and with every other row of the table The self join can be viewed as a join of two copies of the same table. The table is not actually copied, but SQL performs the command as though it were.
The syntax of the command for joining a table to itself is almost the same as that for joining two different tables. To distinguish the column names from one another, aliases for the actual table name are used, since both the tables have the same name. Table name aliases are defined in the FROM clause of the query.

Consider the "EMP" table. Primary key of the "EMP" table is EMPNO. Details of ach employee's manager are just another row in EMP table whose EMPNO is stored in MGR column of some other row. Therefore MGR is a foreign key that refernces EMPNO. TO list out the names of the of the manager with the employee record, one will have to join "EMP" table with itself. 
    SELECT WORKER.ENAME AS ENAME, MANAGER.ENAME AS MANAGER
    FROM EMP WORKER, EMP MANAGER
    WHERE WORKER.MGR = MANAGER.EMPNO;
