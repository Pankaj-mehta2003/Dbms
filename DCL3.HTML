REVOKE COMMAND

REVOKE command removes user access rights or priviliges to the database objects. You can remove priviliges granted to other users by using the REVOKE statement. When you use the REVOKE statement, the priviliges that you specify are revoked from the users name and from any other users to whom those priviliges were granted through the WITH GRANT OPTION clause


SYNTAX: 

    REVOKE [type_of_priviliges] ON [user_name].[table name] FROM [user_name]

EXAMPLE

This command revokes the INSERT permission on the database 'mydb' from 'user1'. 
    REVOKE INSERT ON mydb FROM 'user1'@'localhost';
